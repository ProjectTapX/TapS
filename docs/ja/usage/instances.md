# インスタンス管理

## ステートマシン

```
┌────────┐  start    ┌─────────┐  ready    ┌─────────┐
│stopped │ ────────> │starting │ ────────> │ running │
└────┬───┘           └─────────┘           └────┬────┘
     │                                          │
     │   delete                            stop │
     │                                          v
     │                                     ┌─────────┐
     │                                     │stopping │
     │                                     └────┬────┘
     │                                          │
     │                                          v
     │            ┌─────────┐  exit ≠ 0    ┌────────┐
     │            │ crashed │ <─────────── │stopped │
     │            └─────────┘              └────────┘
     v
   (gone)
```

自動ハイバネーション有効時：

```
running ──idle──> hibernating ──client connect──> starting ──> running
```

## インスタンスの作成

**「インスタンス管理」** → **「新規」** → タイプを選択：
- **テンプレート**（Vanilla / Paper / Purpur / Fabric / Forge / NeoForge）：クイックウィザード + jarの自動ダウンロード
- **Docker**：イメージ / マウント / ポートを自由に指定

フィールドの説明：

| フィールド | 用途 |
|---|---|
| Name | 表示名、重複可 |
| Node | デプロイ先のDaemon（またはグループを選択して自動選択） |
| Working Directory | 空欄 = `<DataDir>/files` ルート。相対パスはそのルートに追加。絶対パスはそのまま使用。Dockerインスタンスの場合、`/data` ボリュームは「Disk Quota」により `<DataDir>/volumes/inst-<short>/` に自動作成され、このフィールドとは無関係 |
| Command | type=dockerの場合：イメージ名（`itzg/minecraft-server`）。セレクターには管理者が設定した表示名が優先表示されます（イメージページで編集可能）。type=genericの場合：実行ファイル |
| Arguments | コマンド引数（配列） |
| Stop Command | stdinに書き込まれるコマンド、例: `stop`（Minecraftの正常シャットダウン） |
| Auto Start | Daemon起動時にこのインスタンスを自動起動するかどうか |
| Auto Restart | クラッシュ後に自動再起動するかどうか |
| Restart Delay | 自動再起動前の待機秒数（デフォルト5） |

## 起動 / 停止

- **起動** / **停止**：stopCmdを呼び出す（デフォルトでstdinに `stop` を書き込み、MCに正常シャットダウンの時間を与える）
- **強制終了**：`docker kill` / `SIGKILL` を直接実行
- **再起動**：停止 → 終了を待機 → 起動（ターミナルで `say` を使ってプレイヤーに通知可能）

## ターミナル

インスタンス詳細ページ → **「ターミナル」**：完全なxterm.jsターミナル、**リアルタイム** stdout、**キーボード** stdin。

権限要件：
- ターミナルを**開く**：`PermView`
- **入力を送信**：`PermTerminal` または `PermControl`

読み取り専用ユーザーはスクロールバック履歴を閲覧可能。キー入力はサイレントに破棄されます。

## ファイル管理

インスタンス詳細ページ → **「ファイル」**タブ：閲覧 / アップロード / ダウンロード / 編集 / 圧縮 / 展開 / リネーム / 移動 / コピー / 削除。

詳細は[ファイルとバックアップ](files.md)を参照してください。

## バックアップ

インスタンス詳細ページ → **「バックアップ」**タブ：

- **バックアップ作成**：作業ディレクトリ全体をzipにし、`backups/<uuid>/<timestamp>.zip` に保存。メモの追加も可能
- **ダウンロード**：zipをエクスポート
- **復元**：作業ディレクトリに展開（既存ファイルを上書き）
- **削除**：単一のバックアップzipを削除

バックアップは**インスタンスのディスククォータ**にカウントされます（マネージドボリューム使用時）。

## モニタリング

インスタンス詳細ページ → **「モニタリング」**タブ：CPU / メモリ / ディスク / ネットワークのリアルタイムグラフ（5秒ごとに1サンプル）。

## プレイヤーリスト

MinecraftインスタンスはSLP自動プローブを使用。詳細ページにオンラインプレイヤー数 + 名前が表示されます。

## インスタンスの削除

インスタンス詳細ページ → **「削除」**：

- コンテナが停止され、`docker rm` が実行される
- 作業ディレクトリ / マネージドボリュームは**保持される**（誤削除を防止するため）。ディスク容量を完全に解放するには、「ファイル管理」から手動で削除してください

## 自動ハイバネーション（Minecraft Java限定）

有効にすると、N分間アイドル後にコンテナが自動停止し、Panelが元のポートでフェイクSLPリスナーを実行します：
- プレイヤーのクライアントサーバーリストにカスタムMOTD + アイコンが表示される
- プレイヤーが接続 → ウェイクをトリガー → 実コンテナが起動 → N秒のカウントダウン後にプレイヤーが完全に参加可能

設定：**「システム設定」** → **「Minecraft Javaサーバー自動ハイバネーション」**でグローバルデフォルト値を設定。インスタンス編集ページ（`hibernationEnabled`、`hibernationIdleMinutes`）で個別にオーバーライド可能。
