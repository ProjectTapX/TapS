# クイックスタート

Minecraftサーバーをゼロから10分で起動する手順です。

## 前提条件

- Panelがデプロイ済みで稼働中であること（[シングルホストデプロイ](../deployment/single-host.md)）
- 少なくとも1つのDaemonノードが追加され、「接続済み」と表示されていること
- ノードのホストにDocker Engineがインストールされていること（Daemonはデフォルトで**Dockerタイプのインスタンスのみ許可**します）

## 1. ログイン

`http://<panel>:24444/` → `admin` / `admin` → パスワードの強制変更。

## 2. 初期設定（システム設定）

ログイン後、**システム設定**に移動し、以下を完了してください：

### 2.1 PanelパブリックURL（必須）

2番目のカード**「PanelパブリックURL」** → Panelの外部URL（例: `https://taps.example.com` または `http://your-ip:24444`）を入力 → 保存。

> これを設定しないと、SSO、ターミナルWebSocketのオリジンチェック、CORSフォールバックがすべて失敗します。

### 2.2 ログインCAPTCHA（強く推奨）

**「ログインCAPTCHA」**カード → **Cloudflare Turnstile** または **reCAPTCHA Enterprise** を選択 → Site Key + Secret を入力 → 「接続テスト」をクリック → 成功後に保存。

> CAPTCHAなしで公開されたPanelは、ブルートフォース攻撃に対してレート制限（デフォルト5回/分）のみに依存します。CAPTCHAを有効にすると、すべてのログインで人間による認証が必要となり、攻撃のハードルが大幅に上がります。

## 3. Dockerイメージのプル

左サイドバー**「イメージ」**ページ → ノードを選択 → **「イメージをプル」**をクリック：

- 一般的なイメージリストから選択（例: `itzg/minecraft-server`、`eclipse-temurin:21-jre`）
- または `repo:tag` を手動入力

プルが完了するまで待ちます（プログレスバーにレイヤーごとのダウンロード状況がリアルタイム表示されます）。

> 管理者はイメージに「表示名」を設定できます（イメージ名の横にある編集アイコンをクリック）。設定すると、インスタンス作成時のランタイムセレクターにはフレンドリー名のみが表示されます。

## 4. インスタンスの作成

### オプションA: テンプレートデプロイ（初心者向け推奨）

**「インスタンス管理」** → **「新規」** → テンプレートを選択：

- **Paper**（推奨 — パフォーマンス + Bukkitプラグインサポート）
- **Vanilla**（公式）
- **Purpur**、**Fabric**、**Forge**、**NeoForge**

入力項目：

| フィールド | 推奨値 | 説明 |
|---|---|---|
| Name | `survival` | 人間が読める識別名 |
| Node | 追加済みのノードを選択 | またはノードグループを選択（最も負荷の低いノードを自動選択） |
| Version | `1.21.4` | Minecraftバージョン |
| Memory Limit | `2G` | Docker `--memory` |
| Disk Quota | `5G` | 5 GiBのループバックマネージドボリュームを自動作成 |
| Host Port | `25565` | 空欄にするとノードのポート範囲から自動割り当て |

**作成してデプロイ**をクリック。

### オプションB: カスタムDockerインスタンス

**「インスタンス管理」** → **「新規」** → タイプ **Docker**：

| フィールド | 例 | 説明 |
|---|---|---|
| Runtime | プル済みイメージから選択 | 表示名が設定されている場合はフレンドリー名を表示 |
| Start Command | `java -Xmx2G -jar server.jar nogui` | コンテナ内で実行されるコマンド |
| Host Port | `25565` | `host:container` 形式 |
| Environment Variables | `EULA=TRUE` | 1行に1つの `KEY=VALUE` |
| Disk Quota | `5G` | 任意。空欄の場合はホストディスクを共有 |

> Daemonはデフォルトで `TAPS_REQUIRE_DOCKER=true` に設定されており、Dockerタイプのインスタンスのみを許可します。非Dockerプロセスを実行するには、Daemonの環境変数を変更してください。（非Dockerインスタンスは本番環境での使用を推奨しません！）

## 5. デプロイの進行状況（テンプレートモード）

進行状況パネルが表示されます：
- サーバーjarの**ダウンロード**（PaperはデフォルトでFastMirrorを使用、中国国内では高速）
- **展開**
- **初回起動**でEULAを自動承認し、ワールドを生成

完了すると、インスタンスの状態が「停止」になります — **起動**をクリックしてください。

## 6. ターミナルの表示

インスタンス詳細ページ → **「ターミナル」**タブ：リアルタイムコンソール + コマンド入力。

## 7. プレイヤーの接続

Minecraftクライアントでサーバーを追加：`<ノードのdisplayHost>:<ホストポート>`、例: `play.example.com:25565`。

ノードエディターで**displayHostが設定されていない**場合、Panelは「アドレス」フィールドのホスト部分にフォールバックします。**外部から到達可能なdisplayHostを必ず設定してください**。設定しないと、プレイヤーに内部IPが表示されます。

---

## 上級: ノードグループ（最も負荷の低いノードを自動選択）

**「ノードグループ」** → グループを作成 → 複数のノードを追加 → インスタンス作成時に「グループ別」を選択 →
Panelがグループ内の各ノードのCPU/メモリ/ディスク負荷をスキャンし、最も負荷の低いノードを選択します。

## 上級: スケジュールタスク

インスタンス詳細ページ → **「スケジュールタスク」** → 作成：

| フィールド | 値 |
|---|---|
| Cron | `0 4 * * *`（毎日午前4:00） |
| Action | `command` / `start` / `stop` / `restart` |
| Content | Action = command の場合、例: `say Server restarting soon` |

## 上級: 自動ハイバネーション（リソース節約）

**「システム設定」** → **「Minecraft Javaサーバー自動ハイバネーション」**でグローバルに有効化。インスタンス編集ページで個別にオーバーライドすることも可能です。

有効にすると：N分間プレイヤーがオンラインでない場合 → 実コンテナを自動停止 → Panelが元のポートで**フェイクSLPリスナー**を実行（プレイヤーのサーバーリストにカスタムMOTD + アイコンが表示される） → プレイヤーが接続すると自動起動し、実コンテナを開始します。
