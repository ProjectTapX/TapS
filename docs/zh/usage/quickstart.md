# 快速上手

10 分钟从零跑起一个 Minecraft 服务器。

## 前置

- Panel 已部署运行（[单机部署](../deployment/single-host.md)）
- 至少一台 Daemon 节点已添加并显示「已连接」
- 节点主机已安装 Docker Engine（Daemon 默认**仅允许 Docker 类型实例**）

## 1. 登录

`http://<panel>:24444/` → `admin` / `admin` → 强制改密。

## 2. 首次配置（系统设置）

登录后进「**系统设置**」，完成以下配置：

### 2.1 Panel 公开地址（必填）

第二张卡片「**Panel 公开地址**」→ 填入 Panel 的对外 URL（如 `https://taps.example.com` 或 `http://your-ip:24444`）→ 保存。

> 不配的话 SSO、终端 WebSocket 同源校验、CORS 回退全部不生效。

### 2.2 登录验证码（强烈推荐）

「**登录验证码**」卡片 → 选择 **Cloudflare Turnstile** 或 **reCAPTCHA Enterprise** → 填入 Site Key + Secret → 点「测试连通性」→ 通过后保存。

> 公网暴露的面板如果不开验证码，暴力破解密码只靠速率限制（默认 5 次/分钟）拦截。开启后每次登录必须通过人机验证，大幅提高攻击门槛。

## 3. 拉取 Docker 镜像

左侧「**镜像**」页 → 选择节点 → 点「**拉取镜像**」：

- 从常用镜像列表中点选（如 `itzg/minecraft-server`、`eclipse-temurin:21-jre`）
- 或手动输入 `repo:tag`

等待拉取完成（进度条会实时显示每层下载状态）。

> 💡 admin 可以给镜像设置"显示名称"（点镜像名旁的编辑图标），设置后在创建实例的运行环境选择器中只显示友好名称。

## 4. 新建实例

### 方式 A：模板部署（推荐新手）

「**实例管理**」→「**新建**」→ 选模板：

- **Paper**（推荐，性能 + Bukkit 插件）
- **Vanilla**（官方原版）
- **Purpur**、**Fabric**、**Forge**、**NeoForge**

填表：

| 字段 | 推荐值 | 说明 |
|---|---|---|
| 名称 | `survival` | 人类可读标识 |
| 节点 | 选已添加的节点 | 或选节点组（自动挑最空的） |
| 版本 | `1.21.4` | Minecraft 版本 |
| 内存上限 | `2G` | Docker `--memory` |
| 磁盘配额 | `5G` | 自动创建 5 GiB loopback 托管卷 |
| 主机端口 | `25565` | 留空自动从节点端口范围分配 |

点 **创建并部署**。

### 方式 B：自定义 Docker 实例

「**实例管理**」→「**新建**」→ 类型选 **Docker**：

| 字段 | 示例 | 说明 |
|---|---|---|
| 运行环境 | 从已拉取的镜像中选择 | 如果设了显示名称只看到友好名 |
| 启动命令 | `java -Xmx2G -jar server.jar nogui` | 容器内执行的命令 |
| 主机端口 | `25565` | `host:container` 格式 |
| 环境变量 | `EULA=TRUE` | 一行一个 `KEY=VALUE` |
| 磁盘配额 | `5G` | 可选；不填则共享宿主机磁盘 |

> ⚠️ Daemon 默认配置 `TAPS_REQUIRE_DOCKER=true`，只允许创建 Docker 类型实例。如需运行非 Docker 进程，需改 daemon 环境变量。 (！！！非 Docker 实例不推荐用于任何生产环境！！！)

## 5. 部署进度（模板方式）

弹出进度面板：
- **下载** server jar（Paper 默认走 FastMirror，国内速度快）
- **解压**
- **首次启动**自动同意 EULA、生成世界

完成后实例状态变为「已停止」，可以点 **启动**。

## 6. 查看终端

实例详情页 →「**终端**」标签：实时控制台 + 命令输入框。

## 7. 玩家连接

Minecraft 客户端添加服务器：`<节点的 displayHost>:<主机端口>`，例如 `play.example.com:25565`。

如果节点编辑里**没填 displayHost**，Panel 会回退用「地址」字段的主机部分。**务必填一个对外可达的 displayHost**，否则会把内网 IP 显示给玩家。

---

## 进阶：节点分组（自动选最空节点）

「**节点分组**」→ 新建分组 → 把多个节点加入 → 创建实例时选「按分组」 →
Panel 会扫描组内每个节点的 CPU/内存/磁盘负载，挑最空的那个创建。

## 进阶：定时任务

实例详情页 →「**定时任务**」→ 新建：

| 字段 | 值 |
|---|---|
| Cron | `0 4 * * *`（每日 4:00） |
| 动作 | `command` / `start` / `stop` / `restart` |
| 内容 | 当动作 = command 时填 `say 服务器即将重启` |

## 进阶：自动休眠（节省资源）

在「**系统设置**」→「**Minecraft Java 服务器自动休眠**」启用全局默认；也可在每个实例编辑页单独覆盖。

启用后：实例无人在线 N 分钟 → 自动停掉真实容器 → Panel 在原端口起一个**假的 SLP 监听器**（玩家在客户端服务器列表里看到自定义 MOTD + 图标） → 一旦玩家点连接，自动唤醒并启动真实容器。
